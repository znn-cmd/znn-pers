# Исправление роутинга для многоязычной страницы аналитики

## ❌ Проблема:
```
Error: getStaticPaths is only allowed for dynamic SSG pages and was found on '/market'.
```

## 🔧 Решение:

### 1. **Создана динамическая страница**
- **Файл**: `src/pages/market/[locale].tsx`
- **URL-адреса**: `/market/en` и `/market/ru`
- **Функции**: `getStaticPaths` и `getStaticProps`

### 2. **Удален статический файл**
- **Удален**: `src/pages/market.tsx` (несовместим с `getStaticPaths`)

### 3. **Добавлен редирект**
- **Файл**: `src/pages/market/index.tsx`
- **Функция**: Автоматический редирект с `/market` на `/market/en`

### 4. **Обновлена навигация**
- **Файл**: `src/components/Header.tsx`
- **Изменено**: Ссылка с `/market` на `/market/en`

## 📁 Структура файлов:

```
src/pages/market/
├── index.tsx          # Редирект с /market на /market/en
└── [locale].tsx       # Динамическая страница для языков
```

## 🔗 URL-адреса:

- **Английский**: `/market/en`
- **Русский**: `/market/ru`
- **Редирект**: `/market` → `/market/en`

## ✅ Результат:

1. **Ошибка исправлена** - `getStaticPaths` теперь используется только в динамической странице
2. **Многоязычность работает** - страницы генерируются для обоих языков
3. **Совместимость сохранена** - старые ссылки `/market` работают через редирект
4. **SEO-оптимизация** - статические страницы для каждого языка

## 🚀 Тестирование:

1. Откройте `/market` - должен произойти редирект на `/market/en`
2. Откройте `/market/en` - английская версия
3. Откройте `/market/ru` - русская версия
4. Переключайте языки через кнопку в хедере

Теперь страница аналитики полностью совместима с Next.js и работает на двух языках! 🎉
