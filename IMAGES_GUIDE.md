## 🎨 Компоненты для изображений

### 1. **PostImage** - Основной компонент
```mdx
<PostImage 
  src="/images/analytics/proptech-landscape.png" 
  alt="Ландшафт проптеха 2025" 
  caption="Распределение инвестиций в проптех по категориям"
  width={800}
  height={600}
/>
```

**Особенности:**
- ✅ Анимация появления (Framer Motion)
- ✅ Поддержка подписей (caption)
- ✅ Оптимизация через Next.js Image
- ✅ Адаптивные размеры
- ✅ Тени и рамки

### 2. **InlineImage** - Для встроенных изображений
```mdx
<InlineImage 
  src="/images/analytics/avm-accuracy.png" 
  alt="Точность AVM моделей" 
  width={500}
  height={300}
/>
```

**Особенности:**
- ✅ Без анимации (быстрая загрузка)
- ✅ Меньший размер
- ✅ Для встраивания в текст
- ✅ Та же оптимизация

## 🎨 Стилизация

### Автоматические стили:
```css
/* Основные изображения */
.prose-custom img {
  @apply rounded-lg shadow-sm my-6 border border-neutral-200;
}

/* Фигуры с подписями */
.prose-custom figure {
  @apply my-8;
}

.prose-custom figcaption {
  @apply text-sm text-neutral-500 text-center mt-3 italic;
}
```

### Визуальные эффекты:
- ✅ **Скругленные углы** (rounded-lg)
- ✅ **Тени** (shadow-sm)
- ✅ **Рамки** (border-neutral-200)
- ✅ **Отступы** (my-6, my-8)
- ✅ **Подписи** (курсив, серый цвет, по центру)

### Next.js Image компонент:
- ✅ **Автоматическое сжатие**
- ✅ **Современные форматы** (WebP, AVIF)
- ✅ **Lazy loading** по умолчанию
- ✅ **Priority loading** для важных изображений

- **PostImage**: 800x600px (основные изображения)

## 📋 Чек-лист для добавления изображений

### 1. Подготовка изображения:
- [ ] Оптимизировать размер (рекомендуется < 500KB)
- [ ] Создать alt-текст для доступности
- [ ] Подготовить подпись (если нужна)

### 2. Размещение файла:
- [ ] Поместить в правильную папку (`/public/images/[category]/`)
- [ ] Использовать понятное имя файла
- [ ] Проверить путь в коде

### 3. Добавление в MDX:
- [ ] Выбрать правильный компонент (PostImage/InlineImage)
- [ ] Указать корректный src путь
- [ ] Добавить alt текст
- [ ] Настроить размеры (width/height)

### 4. Тестирование:
- [ ] Проверить на мобильных устройствах
- [ ] Убедиться в корректной загрузке
- [ ] Проверить alt текст и подписи
